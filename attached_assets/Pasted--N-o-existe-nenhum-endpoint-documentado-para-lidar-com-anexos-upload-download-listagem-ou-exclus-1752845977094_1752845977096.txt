🧩 COMO FUNCIONARÁ O FLUXO NO KEEPTUR:
1. Estrutura para guardar os anexos localmente:
Você precisa ter uma estrutura semelhante a:

bash
Copiar
Editar
/public/uploads/anexos/
└── tarefa_id/
    ├── comprovante.pdf
    ├── contrato-assinado.jpg
    └── orçamento.xlsx
Ou salvar os arquivos em um banco (como MongoDB GridFS, Firebase Storage, S3), desde que armazene a referência associada à tarefa.

2. Modelo de dados (exemplo):
json
Copiar
Editar
{
  "id": "tarefa-123",
  "titulo": "Orçamento Viagem",
  "anexos": [
    {
      "nome": "comprovante.pdf",
      "url": "/uploads/anexos/tarefa-123/comprovante.pdf",
      "tamanho": "1.2MB",
      "tipo": "application/pdf"
    }
  ]
}
3. Na Interface do Keeptur:
Um componente <Anexos /> precisa:

Listar os arquivos com botão de download e exclusão.

Ter um campo para upload de novos arquivos (usando input[type="file"]).

A exclusão precisa chamar seu backend para deletar do local e atualizar a lista.

4. Backend (exemplo Express ou NestJS):
POST /tarefas/:id/anexos → salva arquivo no local correto.

GET /tarefas/:id/anexos → retorna JSON com a lista de anexos.

DELETE /tarefas/:id/anexos/:arquivo → remove o anexo do disco.

⚠️ IMPORTANTE
Como o Monde não suporta exclusão ou upload de anexos via API, se alguém anexar pelo Monde, você só consegue espelhar (ler e exibir), mas não consegue deletar isso de lá.

Para não confundir o usuário, recomendo:

Exibir duas seções separadas:

📥 Anexos adicionados pelo Monde (somente leitura)

📤 Anexos adicionados pelo Keeptur (editáveis e com exclusão)
